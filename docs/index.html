Tailon.

A tiny, dependency-free web viewer that tails and greps your log files — right in the browser.

GitHub Download

No dependencies

The backend is pure Go standard library; the frontend is vanilla HTML/CSS/JS. No frameworks, no npm, no external tail/grep.

Minimal code

Under 1,000 lines of code, total. Small enough to read in one sitting and audit before you run it on a server.

Single binary

Everything — templates, CSS, JS — is embedded. Drop one static binary on a box and run it. Linux & macOS, Intel & ARM.

Just tail & grep

tail follows a file live; grep reads it whole. A regular-expression filter (invertible) narrows either — all done natively in Go.

Group by date

The All files view merges many logs in timestamp order (ISO 8601, syslog, Apache, ctime…), so you read events across hosts chronologically.

Stream over SSE

Lines arrive live over Server-Sent Events and auto-scroll while you're at the bottom — like a terminal, but in a browser tab.

Install

Grab the latest release binary for your OS/arch (installs to /usr/local/bin, or ~/.local/bin):

curl -sL https://raw.githubusercontent.com/tbocek/tailon/main/install.sh | bash

Or build from source with Go 1.26+:

go install github.com/tbocek/tailon@latest

Use it

Point tailon at files, globs, or a whole directory:

tailon /var/log/remote/          # serve a directory, recursively
tailon "/var/log/*.log"          # a glob
tailon -b :9000 access.log        # pick the port

Then open http://localhost:8080, choose a file (or All files), pick tail or grep, and type a regexp to filter.

Recipes

Log rotation? Let logrotate do its job — tailon keeps following a file across rotation and truncation, reopening it automatically.
Many hosts? Ship logs with syslog-ng (or rsyslog) into a directory such as /var/log/remote, point tailon at it, and pick All files — every host's log, merged in timestamp order.

Tailon is a fork of gvalkov/tailon, rebuilt to be tiny and dependency-free. Apache 2.0 licensed.
