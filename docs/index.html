Tailon-ng.

Do one thing, but do it right. A tiny, self-contained web viewer that tails and greps your log files — right in the browser, and nothing else.

GitHub Live demo Download
Test it with your own logs — drag a .log or .gz onto the live demo. Browser-only: nothing is uploaded, there is no backend.

Do one thing, but do it right

Tailing and searching logs is the whole feature list. No dashboards, no ingestion pipeline, no query language. What you get: new lines appear the moment they are written.

Made for self-hosting

No database, no agents, no ingestion pipeline, just files. Point it at the logs already on disk (syslog-ng, journald, Docker, app log files) and read them. One static binary, ideal for a home server or a single VPS.

No built-in auth

Tailon-ng does no authentication of its own. Anyone who can reach its port can read the served files. Bind it to localhost or run it behind an authenticating reverse proxy.

Almost no dependencies

Go standard library throughout, plus two pure-Go decompression libraries (xz, zstd) for reading rotated archives. Vanilla HTML/CSS/JS frontend, no frameworks, no npm, no external tail/grep.

Minimal code

About 2,600 lines of code, total. 1,300 in the Go backend, 1,300 in the browser frontend. Small enough to read in one sitting and audit before you run it on a server.

Single binary

Everything is embedded (templates, CSS, JS). Drop one static binary on a box and run it. Linux & macOS, x86 & ARM.

Just tail & grep — no shell

A single file opens in view — the whole file, following live as it grows. tail follows groups and All files merged; find greps server-side. All with search-as-you-type highlighting, next/prev jumping and editor-style toggles (Aa case, .* regexp, ! invert).

Group by date

The All files view merges many logs in timestamp order (ISO 8601, syslog, Apache, ctime…), so you read events across hosts chronologically.

Rotation-aware

Rotated archives (.gz, .bz2, .xz, .zst, numeric .1, date-stamped, .old, .bak) never pollute live tailing. The gz toggle (in find) lists them in the selector and searches live logs and every archive at once, decoded transparently.

Select, copy, jump

Click a line to highlight it (Ctrl toggles, Shift ranges), then Ctrl-C copies exactly those lines. In merged views, click a line's file prefix to jump into that file's view, scrolled to that very line.

Instant, over SSE

On Linux, appended lines are pushed in milliseconds via inotify (polling fallback elsewhere) and arrive over Server-Sent Events, auto-scrolling while you're at the bottom. Big view and find loads show a 0–100 progress bar; a finished view load jumps to the end.

Append-aware caching

Logs only grow, so the browser caches what it fetched: switching files or modes re-renders instantly and requests only the new bytes. A fully-read archive is never fetched twice.

Not a log platform

Splunk, Datadog, Elastic and Grafana Loki are log platforms: they ingest, index, alert and analyze — and they are very good at it. Tailon-ng deliberately competes with none of that. Install

Grab the latest release binary for your OS/arch (installs to /usr/local/bin, or ~/.local/bin):

curl -sL https://raw.githubusercontent.com/tbocek/tailon-ng/main/install.sh | bash

Or build from source with Go 1.26+:

go install github.com/tbocek/tailon-ng@latest

Or run the multi-arch container image — a distroless static base:

docker run --rm -p 127.0.0.1:8080:8080 -v /var/log:/var/log:ro \
  ghcr.io/tbocek/tailon-ng:latest /var/log

It runs as root so it can read system logs; add --user to drop privileges when the logs allow it (see the README).

Use it

Point tailon-ng at files, directories, or globs — directories are served recursively:

tailon-ng /var/log/remote/                  # a directory, recursively
tailon-ng "/var/log/**.log"                 # recursive glob — quoted, so tailon-ng expands it (new files included)
tailon-ng /var/log/nginx/ /var/log/apache/  # several directories
tailon-ng -b :9000 access.log               # pick the port

Then open http://localhost:8080 (or the port you picked), choose a single file (opens in view: the whole file, following live), a subfolder or All files (tailed and merged), or find — and type a regexp to search.

In the wild: a self-hosted log hub

One tailon-ng tab can stand in for ssh-ing into a fleet of machines. A typical deployment: every service on a self-hosted server logs through the container runtime's syslog driver into a local collector, while other machines on the network ship their logs to that same collector over TLS. The collector (syslog-ng, rsyslog, …) writes one file per source into a shared directory; tailon-ng mounts it read-only and serves it behind a reverse proxy with authentication. Open All files and every host and container becomes a single, live, timestamp-ordered stream you can grep.

Good to know

Log rotation? Let logrotate do its job — tailon-ng keeps following a file across rotation and truncation, reopening it automatically.
Many hosts? Ship logs with syslog-ng (or rsyslog) into a directory such as /var/log/remote, point tailon-ng at it, and pick All files — every host's log, merged in timestamp order.
